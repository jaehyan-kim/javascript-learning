# Chapter 10: 배열 기본 조작과 스프레드 (Basic Array Manipulation and Spread)

## 📖 핵심 개념

자바스크립트에서 배열의 요소를 추가, 삭제, 분할, 병합하기 위한 다양한 배열 인스턴스 메서드(`push`, `pop`, `shift`, `unshift`, `slice`, `splice`)와 스프레드(`...`) 연산자의 동작 원리를 학습합니다.

- **배열 기본 조작과 스프레드 연산자**:
  - 배열의 앞뒤 요소를 조작하거나 대괄호 내부에서 요소를 풀어서 새로운 배열로 조립하는 기법입니다. (예: [용어 사전](../../../../GLOSSARY.md#spread-operator) 참고)
  - **Friendly_Tutor의 친절한 비유**: "배열 조작은 열차를 운행하는 것과 같습니다. 열차의 맨 뒤 칸에 새 칸을 잇는 것(push)이나 분리하는 것(pop), 맨 앞 칸에 기관차를 새로 붙이거나(unshift) 떼어내는 것(shift)과 같습니다. 스프레드 연산자는 샌드위치 식빵 두 장 사이에 여러 재료를 하나하나 꺼내어 차례대로 끼워넣는 과정으로 이해할 수 있습니다."
- **배열 파괴적 메서드의 버그 위험**:
  - `splice`나 `push` 등 원본 배열을 직접 수정하는 파괴적 메서드로 인해 발생하는 React 상태 업데이트 렌더링 누락 버그의 원인과 예방법입니다.
  * **Senior_Practitioner의 실무 팁**: "`splice()`는 원본 배열을 직접 변경(Mutate)하는 대표적인 파괴적 메서드입니다. React와 같은 선언적 프레임워크에서는 상태(State)의 불변성을 유지해야 컴포넌트가 이전 배열과의 참조값 비교를 통해 렌더링을 트리거할 수 있습니다. 만약 `state.splice(...)`를 통해 원본 배열을 직접 칼질하면 주소값이 동일하므로 React는 상태 변화를 감지하지 못해 화면이 갱신되지 않는 렌더링 누락 버그를 겪게 됩니다. 따라서 배열을 가공할 때는 `slice()`와 같은 비파괴 메서드나 스프레드 연산자를 사용하여 항상 새로운 배열을 먼저 생성한 뒤 작업해야 안전합니다."
- **파괴적 vs 비파괴적 메서드 명세**:
  - ECMAScript 스펙에서의 배열 메서드의 두 가지 분류 기준입니다.
  - **PhD_Book_Author의 학술 스펙**: "ECMAScript 사양에서 배열 인스턴스 메서드는 원본 객체인 `this` 내부의 요소와 `length` 속성을 직접 수정하는 **파괴적 메서드(Mutating Methods: push, pop, shift, unshift, splice, sort, reverse)**와, 새로운 Array 인스턴스를 동적으로 생성하여 반환하는 **비파괴적 메서드(Non-mutating Methods: slice, concat, map, filter, flat)**로 구분됩니다. 비파괴적 메서드는 내부적으로 `ArraySpeciesCreate` 추상 연산을 거쳐 새로운 힙 영역에 배열 메모리를 확보한 뒤 요소들을 복사하므로, 원본 데이터의 영속성(Persistence)과 불변성을 유지하는 데 필수적입니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `processQueue(arr)`: 큐(Queue) 동작을 수행합니다. 입력 배열의 맨 앞 원소를 꺼내고, 맨 뒤에 문자열 `'final'`을 추가한 뒤 가공된 결과를 반환합니다.
      - 원본 `arr` 배열이 직접 변형(Mutate)되어야 합니다.
      - 만약 `arr`이 배열이 아니거나 빈 배열이면, 빈 배열을 처리하듯 맨 뒤에 `'final'`만 추가된 배열을 만들어 반환해야 합니다.
  2.  `safeSplice(arr, start, deleteCount)`: 원본 배열 `arr`을 전혀 변경하지 않으면서, `splice`처럼 `start` 인덱스부터 `deleteCount`만큼 요소를 삭제한 새로운 배열을 생성해 반환합니다.
      - 원본 배열 `arr`은 절대 변경되면 안 됩니다.
      - 만약 `arr`이 배열이 아니면 빈 배열 `[]`을 반환합니다.

## 💡 힌트 및 트러블슈팅

- `processQueue`는 원본 배열을 변경해야 하므로 `shift()`와 `push()` 메서드를 직접 사용합니다.
- `safeSplice`는 먼저 스프레드 연산자(`[...arr]`)나 `slice()`로 복사본을 만든 뒤 `splice`를 적용하거나, `slice`의 시작과 끝 범위를 나누어 스프레드로 조합하는 방식으로 원본을 보호할 수 있습니다.
