# Chapter 23: 종합 마일스톤 프로젝트 (Integrated Milestone Project)

## 📖 핵심 개념

종합 마일스톤 프로젝트는 지금까지 개별적으로 학습했던 ES Modules, 비동기 API 통신, 동적 DOM 트리 렌더링, 이벤트 위임 최적화 기술들을 하나로 유기적으로 결합하여 완성형 대시보드 컴포넌트를 조립하는 실무 통합 과정입니다.

- **관심사 분리 (Separation of Concerns)**:
  - 소프트웨어 디자인 원칙 중 하나로, 프로그램을 서로 중복되지 않는 고유한 기능 영역으로 분할하여 제어하는 설계 방식입니다. (예: [용어 사전](../../../../GLOSSARY.md#virtual-dom) 관련 개념 참고)
  - **Friendly_Tutor의 친절한 비유**: "자동차 공장에서 엔진, 바퀴, 브레이크, 운전대를 각각 따로 만들어 둔 상태에서는 차가 달릴 수 없죠. 이 부품들을 제자리에 정확히 고정하고 볼트로 유기적으로 결합하여 최종적인 자율주행 차량(대시보드)으로 완성해 내는 과정과 같습니다."
- **initDashboard**:
  - 대시보드 영역을 청소하고 로딩 상태를 표시한 뒤, 비동기로 수집된 유저 프로필 정보를 토대로 목록 DOM 요소를 동적 렌더링하고 클릭 이벤트를 일괄 통제합니다.
  - **Senior_Practitioner의 실무 팁**: "데이터 조회(fetch), DOM 트리 빌드, 이벤트 리스너 부착, 데이터 삭제 코드가 한 함수에 아무런 격리 없이 떡칠되어 있으면 스파게티 코드가 되어 유지보수가 불가능해집니다. 실무에서는 화면을 그리는 뷰 렌더러 영역과 데이터를 페치해 오는 서비스 영역, 인터랙션을 통제하는 제어 영역으로 역할을 분담해야 합니다. 또한, 삭제 버튼 클릭 시 DOM 노드만 지우는 것이 아니라 상태 객체나 API 백엔드 상태와의 동기화를 항상 염두에 두어야 합니다."
- **통합 컴포넌트의 수명 주기와 메모리 분석**:
  - 비동기 렌더링 컴포넌트가 메모리 상에서 안전하게 GC(가비지 컬렉터)에 수집되도록 구성하기 위한 명세적 접근입니다.
  - **PhD_Book_Author의 학술 스펙**: "대시보드 컴포넌트가 활성화될 때, 렌더링 경로(Rendering Path) 상에 비동기 네트워크 스케줄링(`fetch` 콜백)과 이벤트 위임 바인딩(`addEventListener`)이 순차적으로 결합됩니다. 이때 컴파일러 및 메모리 프로파일러 관점에서, 부모 컨테이너가 DOM 트리에서 소멸할 때 자식 리스너가 정리되지 않으면 부모 요소와의 메모리 참조 포인터가 남아 순환 참조 누수(Leaked Reference)를 만듭니다. 본 마일스톤은 부모 컨테이너에 단 하나의 위임 리스너만 결합함으로써, 컨테이너 소멸 시 하위의 모든 요소와 리스너가 단 한 번의 GC 주기에 물리적으로 소거되도록 명세적으로 최적화되어 있습니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `initDashboard(containerId, userId)`: 지정된 컨테이너에 사용자 할 일 대시보드를 생성하고 렌더링을 제어합니다.
      - `containerId`를 가진 HTML 요소를 탐색합니다. 존재하지 않는 경우 함수 실행을 즉시 종료합니다.
      - API 응답을 받기 전까지 컨테이너 내부에 `<div class="loading">로딩 중...</div>` 메시지를 띄웁니다.
      - fetch API를 사용해 `https://api.example.com/users/${userId}` 로 사용자 정보를 비동기 요청합니다.
      - 네트워크 문제나 통신 실패 발생 시, 컨테이너 내용을 비우고 `<div class="error-msg">에러가 발생했습니다.</div>` 안내를 출력합니다.
      - 응답 성공 시 사용자 이름이 담긴 `<h1>[이름]의 대시보드</h1>`와 할 일 목록 `ul` 구조를 동적으로 만들어 화면에 그립니다. (할 일 목록 아이템 `li` 요소는 `data-id` 속성을 지니며 내부에 삭제 버튼 `<button class="delete-btn">삭제</button>`을 갖도록 조립합니다.)
      - 할 일 목록 `ul` 요소에 클릭 이벤트 위임 리스너를 한 번만 등록합니다. 클릭된 대상이 `button.delete-btn`인 경우, 가장 인접한 `li` 요소를 탐색해 DOM에서 완전히 삭제(`remove()`)합니다.

## 💡 힌트 및 트러블슈팅

- 동적 렌더링 시 HTML 문자열 템플릿 리터럴(Template Literals)을 활용하면 DOM 빌드 가독성을 크게 개선할 수 있습니다.
- 이벤트 위임 시 삭제 버튼에만 반응하도록 `event.target.matches('button.delete-btn')` 분기를 태운 뒤 `li` 요소를 `closest('li')`로 획득하고, `li.remove()`를 실행해 트리에서 노드를 안전하게 배제하십시오.
