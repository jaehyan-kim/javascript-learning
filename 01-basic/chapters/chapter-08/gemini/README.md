# Chapter 08: 화살표 함수와 콜백 기초 (Arrow Functions and Callbacks)

## 📖 핵심 개념

자바스크립트에서 현대적이고 간결하게 함수를 선언하는 **화살표 함수(Arrow Function)** 문법과, 함수의 매개변수로 다른 함수를 전달하여 나중에 실행하는 **콜백 함수(Callback Function)**의 기초를 학습합니다.

- **콜백 함수와 심부름 지시서**:
  - 함수를 다른 함수의 인수로 전달하여 실행 흐름을 제어하는 핵심 기법입니다 (예: [용어 사전](../../../../GLOSSARY.md#lexical-scope) 참고).
  - **Friendly_Tutor의 친절한 비유**: "콜백 함수는 비서에게 건네는 '심부름 작업 지시서'와 같습니다. '필요한 물품을 사 오라'고 지시할 때 구체적으로 어떤 마트에서 무엇을 사올지 적어 비서에게 넘겨주면, 비서가 심부름 도중에 지시서에 적힌 내용을 대신 실행(Callback)하는 방식입니다."
- **화살표 함수의 렉시컬 this 바인딩 버그**:
  - 화살표 함수가 자체적인 `this` 바인딩을 가지지 않아 발생하는 실무 실수입니다.
  - **Senior_Practitioner의 실무 팁**: "화살표 함수는 실행 시점에 dynamic this가 결정되는 일반 함수와 달리, 함수가 선언된 외부 환경의 `this`를 그대로 참조하는 **렉시컬 this 바인딩(Lexical this)**을 가집니다. 따라서 객체의 메서드를 화살표 함수로 정의하면, 메서드 내부의 `this`가 해당 객체가 아니라 전역 객체(`window` 또는 `global`) 혹은 상위 스코프를 가리키게 되어 프로퍼티를 찾지 못하는 오작동 버그가 발생합니다. 객체 메서드를 정의할 때는 화살표 함수 대신 단축 메서드 선언이나 일반 함수 문법을 사용해야 합니다."
- **Construct 내부 메서드의 부재와 new 호출 불가 제약**:
  - 화살표 함수의 명세상 내부적 구조와 생성자 호출 제한 조건입니다.
  - **PhD_Book_Author의 학술 스펙**: "ECMAScript 명세서에 의하면, 일반 함수 객체는 호출 가능한 `[[Call]]`과 생성자로 동작할 수 있는 `[[Construct]]` 내부 메서드를 모두 가집니다. 하지만 **화살표 함수(Arrow Function)**는 **`[[Construct]]` 내부 메서드가 누락**되어 설계되었습니다. 이로 인해 화살표 함수는 `new` 연산자를 사용해 생성자 함수로 호출할 경우 `TypeError`를 발생시키며, 객체를 생성하기 위한 프로토타입(`prototype`) 프로퍼티 역시 생성되지 않아 메모리 구조적으로 경량화된 특징을 지닙니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `processNumbers(arr, callback)`: 배열의 각 요소를 콜백 함수에 통과시킨 가공된 새로운 배열을 반환합니다.
      - 매개변수 `arr`이 배열이 아니거나 `callback`이 함수가 아닌 경우 빈 배열 `[]`을 반환합니다.
      - `arr` 내의 각 원소들을 하나씩 순회하며 `callback` 함수에 전달해 실행한 반환값들을 모아 새 배열로 만들어 반환합니다.
      - 화살표 함수와 고차 함수 문법 또는 일반 루프를 적절히 조화하여 작성해 보십시오.

## 💡 힌트 및 트러블슈팅

- `processNumbers` 내에서 `Array.isArray(arr)`와 `typeof callback === 'function'` 조건으로 입력 검증을 수행할 수 있습니다.
- 배열 각 요소를 순회하여 새 배열을 생성할 때는 내장 메서드인 `arr.map(callback)` 또는 `for` 루프와 `push` 메서드를 사용할 수 있습니다.
