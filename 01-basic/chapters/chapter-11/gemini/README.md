# Chapter 11: 배열 순회와 고차 함수 기초 (Array Iteration and Higher-Order Functions)

## 📖 핵심 개념

자바스크립트에서 다른 함수를 인수로 전달받는 **고차 함수(Higher-Order Function)**의 구조와, 배열의 요소를 편리하게 가공 및 필터링할 수 있는 배열의 핵심 고차 함수(`forEach`, `map`, `filter`)를 학습합니다.

- **배열 순회와 고차 함수**:
  - 배열 순회를 루프 대신 함수 단위의 파이프라인으로 전환하여 가독성과 격리성을 극대화하는 기법입니다. (예: [용어 사전](../../../../GLOSSARY.md#lexical-scope) 참고)
  - **Friendly_Tutor의 친절한 비유**: "고차 함수와 배열 순회는 컨베이어 벨트가 설치된 제조 공장 생산 라인과 같습니다. 벨트 위를 흘러가는 원자재 전체를 전수 조사하는 것(forEach), 모든 자재를 기계로 깎아 가공품으로 변환하는 것(map), 규격에 맞지 않는 불량품이나 홀수 부품만 걸러내고 버리는 것(filter)으로 비유할 수 있습니다."
- **forEach 부수 효과와 map/filter의 불변성**:
  - `forEach` 순회 도중 발생하는 Side Effect와, 새로운 배열을 매번 반환하는 `map`/`filter`를 이용해 불변성을 유지하는 실무 팁입니다.
  * **Senior_Practitioner의 실무 팁**: "`forEach`는 단순히 요소를 순회하는 루프로서, 반환값(`undefined`)이 없습니다. 따라서 어떤 연산 결과를 모으려면 콜백 내부에서 상위 스코프의 외부 전역 변수나 배열을 직접 수정해야 하므로 코드의 결합도가 높아지고 **부수 효과(Side Effect)**가 발생하기 쉽습니다. 반면 `map`과 `filter`는 항상 원본 배열을 수정하지 않고 새로운 가공 배열을 반환하는 비파괴 고차 함수입니다. 실무에서는 버그 예방과 디버깅 용이성을 위해 가능한 한 외부 상태를 조작하지 않는 순수 함수 형태의 `map`/`filter` 체이닝을 사용하는 것이 권장됩니다."
- **고차 함수의 콜백 실행 컨텍스트와 순수 함수**:
  - 고차 함수의 콜백 실행 구조 및 호출 시 만족해야 하는 함수형 패러다임의 특징입니다.
  - **PhD_Book_Author의 학술 스펙**: "자바스크립트의 배열 고차 함수(`map`, `filter`, `forEach` 등)는 첫 번째 인자로 전달받은 콜백 함수를 호출할 때, 각 요소의 값, 인덱스, 그리고 순회 중인 배열 자체(`thisArg`가 지정되지 않은 경우)를 인자로 바인딩하여 렉시컬 실행 컨텍스트를 생성합니다. 이때 인자로 활용되는 콜백 함수가 **순수 함수(Pure Function)**가 되기 위해서는 두 가지 ECMAScript 명세 동작을 충족해야 합니다. 첫째, 동일한 입력에 대해 항상 동일한 값을 반환할 것(Referential Transparency). 둘째, 실행 도중 평가 엔진 외부의 상태를 변경하는 `Side Effect`를 일으키지 않을 것입니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `getOddSquares(arr)`: 숫자 배열을 받아 홀수만 걸러내고(filter), 그 홀수들을 각각 제곱한(map) 새로운 배열을 반환합니다.
      - 원본 `arr` 배열은 변경되면 안 됩니다.
      - 만약 `arr`이 배열이 아니거나 빈 배열이면 빈 배열 `[]`을 반환합니다.
      - `filter`와 `map` 고차 함수를 체이닝하여 구현해 보십시오.

## 💡 힌트 및 트러블슈팅

- 홀수를 판별하는 식은 `num % 2 !== 0`을 사용할 수 있습니다. 음수 홀수까지 올바르게 걸러낼 수 있도록 주의합니다.
- 고차 함수 체이닝 예시: `arr.filter(num => ...).map(num => ...)`
