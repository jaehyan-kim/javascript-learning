# Chapter 12: 배열 누적과 정제 심화 (Advanced Array Reduction and Sorting)

## 📖 핵심 개념

배열의 요소를 검색하는 `find`, 조건 만족 여부를 검증하는 `some`/`every` 메서드와, 전체 데이터를 누적하여 단일 값으로 정제하는 `reduce`, 그리고 배열의 정렬을 처리하는 `sort` 메서드의 심화 동작 원리를 학습합니다.

- **배열 누적과 검색/정렬 심화**:
  - 배열 내 요소를 검색하거나 전체 요소의 속성 합을 계산하고 정렬하는 심화 연산 기법입니다.
  - **Friendly_Tutor의 친절한 비유**: "배열 누적과 정제는 쇼핑을 마친 뒤 영수증을 처리하는 것과 같습니다. 장바구니에 담긴 모든 상품의 가격을 차례대로 더해 하나의 총합을 구하는 과정은 `reduce`와 같으며, 수많은 지원자 카드 중에서 합격자 한 명을 찾아내는 것은 `find`, '합격자가 단 한 명이라도 있는가?' 검사하는 것은 `some`에 비유할 수 있습니다."
- **sort() 정렬 오류와 원본 오염 버그**:
  - `sort()` 메서드 호출 시 비교 함수를 전달하지 않거나 원본이 오염됨으로써 유발되는 실무 버그와 대처법입니다.
  * **Senior_Practitioner의 실무 팁**: "자바스크립트의 `sort()`는 기본적으로 요소를 **문자열로 형변환한 뒤 UTF-16 유니코드 코드 포인트 순**으로 정렬합니다. 이로 인해 숫자 배열 `[1, 10, 2]`를 정렬하면 `[1, 2, 10]`이 아닌 `[1, 10, 2]` 순서로 정렬되는 치명적인 오작동 버그가 발생합니다. 또한 `sort()`는 원본 배열을 오염시키는 **파괴적 메서드**이므로 원본을 보존해야 하는 프론트엔드 아키텍처에서는 반드시 스프레드나 `slice()` 등으로 복사본을 만든 뒤, `sort((a, b) => a - b)`와 같이 명시적인 비교 함수를 제공해 정렬해야 합니다."
- **reduce 누산기 작동 방식과 sort 비교 명세**:
  - `reduce` 초기값 생략 시의 예외 조건 및 `sort` 비교 함수의 기저 스펙입니다.
  - **PhD_Book_Author의 학술 스펙**: "ECMAScript 명세에 따르면, `reduce` 메서드는 초기값(initialValue) 제공 여부에 따라 동작이 분기됩니다. 초기값이 있으면 인덱스 0부터 누산기(accumulator)가 계산을 시작하며, 없으면 인덱스 0의 값을 첫 누산기로 지정하고 인덱스 1부터 순회를 시작하므로 빈 배열에 초기값 없이 호출 시 `TypeError`가 던져집니다. 한편, `Array.prototype.sort`에 전달되는 `compareFunction(a, b)`은 반환값의 부호에 의해서 정렬 방향이 결정됩니다. 반환값이 `0`보다 작으면 `a`를 `b`보다 낮은 인덱스로, `0`보다 크면 `b`를 `a`보다 낮은 인덱스로 재배치하며, 일관성 없는 비교 함수는 정렬 알고리즘 내부 정합성을 깨뜨릴 수 있습니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `findSpecialUser(users)`: 객체 배열 `users`에서 `age` 속성이 `18` 이상인 첫 번째 유저 객체를 반환합니다.
      - 조건에 부합하는 유저가 없으면 `undefined`를 반환합니다.
      - 만약 `users`가 배열이 아니거나 빈 배열인 경우 `undefined`를 반환합니다.
  2.  `calculateTotalPoint(orders)`: 주문 객체 배열 `orders` 내의 모든 `order` 객체의 `point` 값을 합산한 최종 포인트를 반환합니다.
      - `reduce` 메서드를 반드시 사용하여 구현하십시오.
      - `orders`가 배열이 아니거나 빈 배열인 경우 `0`을 반환합니다.
      - 각 `order`는 `{ point: number, ... }` 구조를 가집니다. point 값이 누락되었거나 숫자가 아닌 경우 해당 order의 포인트는 `0`으로 처리하십시오.
  3.  `sortNumbers(numbers)`: 숫자 배열을 오름차순으로 정렬하여 새로운 배열로 반환합니다.
      - `sort` 메서드와 명시적인 비교 함수를 반드시 사용해야 합니다.
      - 원본 배열 `numbers`가 변경되지 않도록 복사본을 만들어 정렬해야 합니다.
      - `numbers`가 배열이 아니거나 빈 배열인 경우 빈 배열 `[]`을 반환합니다.

## 💡 힌트 및 트러블슈팅

- `findSpecialUser`는 `Array.prototype.find` 메서드를 활용하여 간결하게 조건식을 작성할 수 있습니다.
- `calculateTotalPoint`는 `typeof order.point === 'number' ? order.point : 0` 조건식을 활용해 값이 숫자일 때만 누적되도록 방어적으로 작성하십시오. `order.point || 0` 방식은 문자열 타입의 숫자가 유입될 때 예기치 않은 결합 버그를 낼 수 있습니다.
- `sortNumbers`는 `[...numbers]` 또는 `numbers.slice()`를 통해 원본 배열을 복제한 뒤에 `.sort((a, b) => a - b)` 체이닝을 적용합니다.
