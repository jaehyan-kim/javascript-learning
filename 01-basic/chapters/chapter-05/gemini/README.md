# Chapter 05: 조건문 분기 (Conditional Branching)

## 📖 핵심 개념

자바스크립트에서 조건에 따라 프로그램의 흐름을 다르게 실행하도록 제어하는 조건문과 연산자에 대해 학습합니다.

- **조건문 (Conditional Statement)**:
  - 조건에 따라 코드 블록의 실행 여부를 제어하는 구문입니다 (예: [용어 사전](../../../../GLOSSARY.md) 참고).
  - **Friendly_Tutor의 친절한 비유**: "조건문은 철도 선로의 분기점에 설치된 '선로 전환기'와 같습니다. 기차가 도달했을 때 전환기 방향에 따라 기차가 갈 선로가 결정되듯이, 자바스크립트 엔진도 조건식의 평가 결과에 따라 서로 다른 실행 흐름(코드 블록)으로 분기합니다."
- **switch case와 break**:
  - 동일한 대상을 여러 값과 비교할 때 가독성이 높은 분기문입니다.
  - **Senior_Practitioner의 실무 팁**: "switch 문 내 각 case 블록의 마지막에 `break`를 빠뜨리면, 일치하는 case 다음의 모든 case 블록들까지 이어서 실행되는 **fall-through(폴스루) 현상**이 발생합니다. 이는 실무에서 매우 흔하게 발생하는 논리 버그이므로 특별한 의도가 없다면 반드시 각 case마다 `break`를 배치해야 합니다."
- **엄격한 비교와 삼항 연산자 식**:
  - 자바스크립트 엔진이 switch case를 비교할 때의 특징과 삼항 연산자의 평가 특성입니다.
  - **PhD_Book_Author의 학술 스펙**: "ECMAScript 명세에 따르면 `switch` 문은 비교 시 **엄격한 일치 비교(Strict Equality Comparison, `===`)** 알고리즘을 사용하므로, 타입 변환 없이 값과 타입이 모두 같아야 매칭됩니다. 또한 `if` 문은 값을 만들지 않는 **문(Statement)**이지만, 삼항 연산자(`? :`)는 평가되어 특정한 값을 가지는 **식(Expression)**으로 취급됩니다. 따라서 삼항 연산자는 변수 대입이나 함수의 리턴 위치 등 식을 요구하는 자리에 바로 대입될 수 있습니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `evalGrade(score)`: 입력된 점수에 부합하는 학점 문자를 반환합니다.
      - 점수(`score`)가 90점 이상이면 `'A'`를 반환합니다.
      - 점수(`score`)가 80점 이상 90점 미만이면 `'B'`를 반환합니다.
      - 점수(`score`)가 70점 이상 80점 미만이면 `'C'`를 반환합니다.
      - 점수(`score`)가 70점 미만이거나, 유효하지 않은 입력(숫자가 아니거나 0 미만 혹은 100을 초과하는 숫자)인 경우 `'F'`를 반환합니다.
  2.  `checkAccess(age, hasLicense)`: 나이와 면허 소지 상태에 따른 접근 여부를 반환합니다.
      - `age`가 18세 이상이고 `hasLicense`가 `true`인 경우 `'granted'`를 반환합니다.
      - 그 외의 모든 경우는 `'denied'`를 반환합니다.
      - 단, 이 함수는 반드시 **삼항 연산자**를 사용하여 단일 리턴문으로 작성해야 합니다.
  3.  `getRoleName(roleCode)`: 권한 코드에 따른 역할 이름을 반환합니다.
      - `roleCode`가 `'ADMIN'`이면 `'관리자'`를 반환합니다.
      - `roleCode`가 `'EDITOR'`이면 `'편집자'`를 반환합니다.
      - `roleCode`가 `'USER'`이면 `'일반 사용자'`를 반환합니다.
      - 그 외의 모든 경우는 `'알 수 없는 역할'`을 반환합니다.
      - 단, 이 함수는 반드시 **switch** 문을 사용해야 합니다.

## 💡 힌트 및 트러블슈팅

- `evalGrade`에서는 입력값이 유효한 숫자형인지 `typeof score === 'number'`로 확인하고 `isNaN(score)`을 활용해 걸러내는 방어적 코드 작성이 유용합니다.
- `checkAccess`에서는 조건들을 논리곱 연산자(`&&`)로 결합하여 하나의 삼항 연산자에 적용할 수 있습니다.
- `getRoleName`에서는 각 `case` 블록 마지막에 `break`를 작성하거나, 함수인 점을 이용해 `return`을 사용해 fall-through를 방지하세요.
