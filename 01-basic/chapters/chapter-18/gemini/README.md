# Chapter 18: 이벤트 리스너와 이벤트 객체 (Event Listeners and Event Object)

## 📖 핵심 개념

DOM 요소에서 발생하는 이벤트를 가로채어 제어하는 방법과, 브라우저의 기본 새로고침 동작을 중단시켜 웹 애플리케이션의 데이터와 상태를 안전하게 보존하는 이벤트 핸들링 기법을 배웁니다.

- **이벤트 리스너 (addEventListener)**:
  - DOM 요소에서 발생하는 이벤트(클릭, 제출, 키보드 입력 등)를 모니터링하다가 이벤트 발생 시 핸들러 함수를 실행하도록 바인딩하는 메커니즘입니다 ([용어 사전](../../../../GLOSSARY.md) 참고).
  - **Friendly_Tutor의 친절한 비유**: "누군가 현관 벨을 누르면 작동하는 초인종 센서와 벨을 누른 사람의 정보(방문 시간, 누른 버튼 등)를 분석하는 과정과 같습니다."
- **preventDefault()와 SPA 상태 소실 방지**:
  - 브라우저가 특정 이벤트에 대해 실행하는 기본 행동(Default Action)을 취소하는 메서드입니다.
  - **Senior_Practitioner의 실무 팁**: "`<form>` 제출(submit)이나 `<a>` 태그 클릭 시 브라우저는 기본적으로 페이지를 강제 리로드(Reload)하여 새로운 요청을 보냅니다. 이는 싱글 페이지 애플리케이션(SPA)이나 리액트 환경에서 메모리 상에 유지되던 모든 상태(State)와 임시 데이터를 즉시 초기화하여 소실시키는 치명적인 결함으로 이어집니다. 따라서 자바스크립트로 화면을 동적 갱신할 때는 반드시 이벤트 핸들러 시작 부분에서 `event.preventDefault()`를 호출하여 브라우저의 기본 페이지 이동/새로고침 동작을 중단해야 합니다."
- **W3C DOM Event 명세 및 디폴트 액션 취소 플래그**:
  - 브라우저의 이벤트 전파 및 기본 동작 취소의 내부 작동 방식입니다.
  - **PhD_Book_Author의 학술 스펙**: "W3C DOM Event 및 HTML Living Standard 명세에 따르면, 이벤트가 트리 상에서 전파되는 동안 `Event.preventDefault()`가 호출되면 이벤트 객체의 `canceled flag`가 `true`로 설정됩니다. 이벤트 리스너 실행이 끝난 후 브라우저는 이 플래그가 설정되어 있는지를 확인하며, `true`로 지정되어 있다면 해당 이벤트에 매핑된 사용자 에이전트의 기본 동작(Default Action - 예: 폼 제출에 따른 네비게이션)을 취소(Cancel)하고 실행하지 않습니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `setupFormSubmit(formId, callback)`: 주어진 ID를 가진 `<form>` 요소를 조회하여 `submit` 이벤트 리스너를 추가합니다.
      - `formId`를 사용하여 해당 폼 요소를 안전하게 조회합니다. 만약 폼 요소가 존재하지 않으면 리스너 바인딩을 수행하지 않고 중단합니다.
      - `submit` 이벤트 핸들러가 트리거되면 반드시 `event.preventDefault()`를 먼저 호출하여 브라우저의 기본 리로드 동작을 방지합니다.
      - 폼 내부 입력 필드들의 데이터를 `FormData`와 `Object.fromEntries` API를 활용하여 `{ [name]: value }` 형태의 단순 객체로 가공합니다.
      - 가공된 데이터 객체를 매개변수로 담아 `callback` 함수를 실행합니다.

## 💡 힌트 및 트러블슈팅

- `submit` 이벤트는 `event.currentTarget`을 참조하여 이벤트를 수신한 폼 요소 자체를 획득할 수 있습니다.
- 폼 내부의 입력 요소들은 반드시 `<input name="username">`과 같이 `name` 속성을 지녀야 `FormData` 객체에서 정상적으로 값을 수집할 수 있습니다.
