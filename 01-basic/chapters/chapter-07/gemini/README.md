# Chapter 07: 함수의 기본과 매개변수 (Functions and Parameters)

## 📖 핵심 개념

자바스크립트에서 재사용 가능한 코드 블록인 **함수**를 정의하는 방법(선언문 vs 표현식)과 매개변수를 유연하게 제어하는 기법(기본 매개변수, 나머지 매개변수)에 대해 학습합니다.

- **매개변수와 요리 주문 옵션**:
  - 함수를 호출할 때 외부에서 전달하는 입력값(매개변수)과 이를 제어하는 기법을 배웁니다 (예: [용어 사전](../../../../GLOSSARY.md#hoisting) 참고).
  - **Friendly_Tutor의 친절한 비유**: "함수는 레시피에 따라 동작하는 주방장과 같고, 매개변수는 요리 주문서의 선택 옵션과 같습니다. '매개변수 기본값'은 고객이 토핑 옵션을 선택하지 않았을 때 들어가는 기본 재료이며, '나머지 매개변수'는 어떤 양의 재료든 상관없이 한꺼번에 담아 전달하는 재료 상자와 같습니다."
- **기본값 누락과 undefined 프로퍼티 참조 에러**:
  - 함수 매개변수 기본값 설정 미비로 발생하는 런타임 오류입니다.
  - **Senior_Practitioner의 실무 팁**: "자바스크립트 함수는 선언된 매개변수 개수보다 적은 인자로 호출되어도 에러를 발생시키지 않고 누락된 매개변수에 `undefined`를 바인딩합니다. 이때 함수 내부에서 기본값 설정 없이 이 매개변수의 프로퍼티나 메서드(예: `title.length`)를 참조하면, `Cannot read properties of undefined`와 같은 치명적인 런타임 에러가 발생해 서비스가 중단됩니다. 이를 예방하기 위해 ES6 기본 매개변수(Default Parameter)를 적극 활용해 방어적으로 코드를 작성해야 합니다."
- **함수 인스턴스화 시점과 arguments vs Rest Parameter**:
  - 선언문과 표현식의 호이스팅 메커니즘 차이 및 두 가변 인자 제어 방식의 차이점입니다.
  - **PhD_Book_Author의 학술 스펙**: "엔진이 실행 컨텍스트를 생성할 때, **함수 선언문**은 평가 단계에서 식별자 등록과 동시에 함수 객체 인스턴스를 메모리에 할당(Instantiation)하므로 선언 전 호출이 가능합니다. 반면 **함수 표현식**은 변수 호이스팅 규칙을 따르므로 평가 단계에서는 변수 식별자만 등록되고 초기화되지 않거나 `undefined` 상태로 대기하며, 런타임에 해당 할당문 실행 흐름에 도달해야 함수 인스턴스가 생성됩니다. 또한, 유사 배열(Array-like) 객체이자 호출 시마다 생성되어 성능 오버헤드를 일으키는 `arguments` 객체와 달리, **나머지 매개변수(Rest Parameter, `...`)**는 자바스크립트의 표준 Array 인스턴스이므로 `map`, `reduce` 등 고차 함수 메서드를 즉시 사용할 수 있다는 차이가 있습니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `greetUser(name, title)`: 환영 메시지 문자열을 반환합니다.
      - `안녕하세요, [name] [title]님!` 형식의 문자열을 반환합니다.
      - `title` 매개변수가 생략되었거나 `undefined`가 전달된 경우 기본값으로 `'회원'`을 지정하여 반환해야 합니다. (ES6 Default Parameter 필수 적용)
      - `name` 매개변수가 생략되었거나 `undefined`가 전달된 경우 기본값으로 `'손님'`을 지정하여 반환해야 합니다.
  2.  `sumAll(...nums)`: 임의의 개수로 입력되는 모든 인자들의 합을 계산하여 반환합니다.
      - 반드시 나머지 매개변수(Rest Parameter) 문법을 활용하십시오.
      - 매개변수가 아무것도 전달되지 않은 경우에는 `0`을 반환합니다.

## 💡 힌트 및 트러블슈팅

- `greetUser`는 ES6 백틱(` `) 템플릿 리터럴을 사용하여 `안녕하세요, ${name} ${title}님!` 형태로 구현하면 깔끔합니다.
- `sumAll`은 `...nums`를 배열로 인식하므로, 배열의 합을 구하는 `reduce` 메서드나 `for...of` 루프를 사용해 합계를 구할 수 있습니다.
