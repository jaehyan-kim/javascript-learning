# Chapter 06: 반복문 제어 (Loop Control)

## 📖 핵심 개념

자바스크립트에서 동일하거나 유사한 작업을 반복 수행하는 반복문(`for`, `while`)과 실행 흐름을 제어하는 제어 흐름문(`break`, `continue`)을 학습합니다.

- **반복문과 컨베이어 벨트**:
  - 특정 조건을 만족하는 동안 코드 블록을 반복적으로 수행하게 만드는 제어문입니다 (예: [용어 사전](../../../../GLOSSARY.md) 참고).
  - **Friendly_Tutor의 친절한 비유**: "반복문은 공장의 '컨베이어 벨트'와 유사합니다. 벨트 위로 지나가는 수많은 제품들 중에서 불량품이 나오면 `continue`를 눌러 건너뛰고, 치명적인 결함이 발견되면 `break`를 눌러 벨트 전체를 즉시 멈추는 것과 같이, 반복문 제어를 통해 대량의 데이터를 효율적으로 검사하고 가공할 수 있습니다."
- **무한 루프와 브라우저 크래시**:
  - 반복 종료 조건이 잘못 설계되어 영원히 반복을 멈추지 않는 상황입니다.
  - **Senior_Practitioner의 실무 팁**: "`while` 문을 설계할 때 탈출 조건식의 변수 값을 업데이트하는 코드를 빠뜨리거나 잘못 조건식을 잡으면 **무한 루프(Infinite Loop)**에 빠지게 됩니다. 이는 CPU 점유율을 100%로 치솟게 만들어 싱글 스레드로 동작하는 브라우저의 렌더링 스레드를 차단하고, 결국 브라우저 탭 전체가 먹통이 되는 크래시(Crash) 장애를 유발합니다. 따라서 반복 조건과 증감 식의 흐름을 꼼꼼하게 검증해야 합니다."
- **for 루프와 while 루프의 평가 절차**:
  - ECMAScript 명세서 기준 두 반복문 동작 방식의 차이점입니다.
  - **PhD_Book_Author의 학술 스펙**: "ECMAScript 명세상 `for` 문(`for (initialize; test; increment) statement`)은 먼저 초기화 식을 단 한번 평가한 뒤, 매 반복 단계마다 `test` 평가 -> `statement` 실행 -> `increment` 평가 순서를 밟습니다. 반면 `while (expression) statement` 문은 단지 매 루프마다 `expression`을 평가하고, 그 결과가 Truthy일 때만 `statement`를 실행합니다. 특히 `continue`를 사용했을 때, `for` 루프는 증감문(`increment`)으로 이동해 루프 카운터를 변경하지만, `while` 루프는 조건 평가(`expression`) 단계로 바로 건너뛰기 때문에 루프 제어 변수가 업데이트되지 않아 무한 루프에 걸리기 쉬운 내부 상태 차이가 존재합니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `sumNumbers(n)`: 1부터 `n`까지의 자연수 합을 계산하여 반환합니다.
      - 매개변수 `n`이 자연수가 아니거나 0 이하의 값이면 `0`을 반환합니다.
      - `for` 또는 `while` 루프를 사용하여 구현해야 합니다.
  2.  `findFirstEven(arr)`: 배열 `arr`을 순회하며 가장 처음으로 나타나는 짝수를 찾아 반환합니다.
      - 배열 내 짝수를 찾은 즉시 반복을 멈추고 해당 값을 조기에 반환(Early Return/Break)해야 합니다.
      - 짝수를 찾지 못했거나 매개변수가 유효한 배열이 아니면 `undefined`를 반환합니다.

## 💡 힌트 및 트러블슈팅

- `sumNumbers`에서는 `typeof n !== 'number'` 검사와 정수 여부를 판별하는 `Number.isInteger(n)`를 사용해 비정상적인 인자를 사전 차단하는 것이 좋습니다.
- `findFirstEven`에서는 배열을 처음부터 순회하면서 `element % 2 === 0` 조건을 만났을 때 바로 해당 요소를 `return`하면 자연스럽게 조기 탈출이 구현됩니다.
