# Chapter 09: 객체 리터럴과 참조 데이터 (Object Literals and Reference Data)

## 📖 핵심 개념

자바스크립트에서 객체를 정의하는 기본 방식인 객체 리터럴과, 참조형 데이터(Reference Type)가 메모리 상에서 어떻게 주소를 공유하고 동작하는지 학습합니다. 또한 이를 안전하게 복사하고 가공하기 위한 구조 분해 할당(Destructuring) 문법의 활용법을 다룹니다.

- **참조형 데이터**:
  - 원시 타입과 달리 메모리 상의 실제 주소값(참조값)을 저장하는 데이터 타입입니다. (예: [용어 사전](../../../../GLOSSARY.md#shallow-vs-deep-copy) 참고)
  - **Friendly_Tutor의 친절한 비유**: "참조형 데이터는 동일한 구글 문서(주소)를 공유하는 것과 같습니다. 두 사람이 각자의 컴퓨터에서 같은 구글 문서의 URL을 열고 편집하면, 한 사람이 글을 지웠을 때 다른 사람의 화면에서도 글이 사라지는 것처럼 원본 문서가 직접 수정됩니다."
- **구조 분해 할당**:
  - 객체나 배열을 해체하여 그 값을 개별 변수에 할당하는 문법입니다. (예: [용어 사전](../../../../GLOSSARY.md#destructuring-assignment) 참고)
  - **Senior_Practitioner의 실무 팁**: "단순히 `const newUser = user;`와 같이 객체 참조를 단순 복사한 뒤, `newUser.name = newName`으로 속성을 변경하면 원본 객체의 데이터까지 오염됩니다. 이는 리액트의 상태 변경 감지를 방해해 렌더링 누락을 유발하거나, 공유 서비스 메모리의 상태값을 오염시켜 결제 정보가 오작동하는 등의 대규모 장애로 이어집니다. 따라서 객체를 수정할 때는 반드시 구조 분해 할당이나 스프레드 연산자(`...`) 등을 사용하여 새로운 객체 복사본을 생성하고 변경을 적용해야 합니다."
- **참조 데이터 메모리 메커니즘**:
  - 자바스크립트 엔진의 Call Stack과 Memory Heap 구조 하에서 참조형 변수의 저장 구조 및 얕은 복사 특성입니다.
  - **PhD_Book_Author의 학술 스펙**: "자바스크립트 엔진은 원시 타입 변수를 Call Stack 영역에 값 자체를 바인딩하여 관리하는 반면, 객체와 같은 참조형 데이터는 Memory Heap 영역에 메모리를 할당하고 Call Stack의 변수에는 힙 메모리의 포인터(참조 주소)를 저장합니다. 구조 분해 할당(`{ ...user }`)이나 `Object.assign`은 객체의 최상위 프로퍼티만 새 주소에 복사하는 **얕은 복사(Shallow Copy)**를 수행하므로, 1단계 프로퍼티는 복사되지만 중첩된 객체 프로퍼티는 원본 객체와 힙 메모리 상에서 여전히 동일한 주소를 공유하므로 완전한 독립이 되지 않음을 주의해야 합니다."

## 🧪 실습 미션

- **미션 파일**: `mission.js`
- **요구 사항**:
  1.  `cloneAndModifyUser(user, newName)`: user 객체를 구조 분해 할당을 통해 새로운 복사본을 생성하고 name을 newName으로 교체하여 반환합니다.
      - 원본 `user` 객체는 절대 오염되지 않아야(Mutate되지 않아야) 합니다.
      - 매개변수 `user`가 객체가 아니거나 null인 경우 빈 객체 `{}`에 `name: newName`을 설정하여 반환합니다.
      - 구조 분해 할당을 적극적으로 활용하여 새 객체를 빌드해 보십시오.

## 💡 힌트 및 트러블슈팅

- 원본 객체 오염 여부는 `const updated = cloneAndModifyUser(user, 'Jane'); updated !== user`와 같이 참조값 비교를 통해 검증할 수 있습니다.
- `const { name, ...rest } = user || {};`와 같은 구조 분해 할당 문법을 사용하여 `name`을 제외한 나머지 프로퍼티를 분리한 뒤 새 객체로 조합할 수 있습니다.
