# Chapter 05: TypeScript 제네릭(Generic) 기초 (Introduction to Generics)

## 📖 핵심 개념
재사용성과 타입 안전성을 동시에 극대화할 수 있는 TypeScript의 제네릭(Generic)의 핵심 원리와 정의 방식을 학습합니다.

*   **제네릭 (Generic)**:
    *   타입을 인수로 받아서 클래스나 함수 내부의 속성 및 반환 타입을 동적으로 결정하는 다형성 설계 기법입니다. (예: [용어 사전](../../../../GLOSSARY.md#generics) 참고)
    *   **Friendly_Tutor의 친절한 비유**: "안에 무엇이든 자유롭게 담을 수 있지만, 한 번 물건을 담으면 그 내용물의 형태와 특징(타입)을 100% 보존하고 기억하는 특수 투명 밀폐 용기와 같습니다."
*   **타입 매개변수 제약조건 (Constraints)**:
    *   `extends` 키워드를 사용하여 제네릭 타입 변수가 특정 인터페이스나 기본 형태를 만족하도록 강제하는 방법입니다.
    *   **Senior_Practitioner의 실무 팁**: "제네릭 타입을 단순한 `any` 타입의 대체제처럼 남용하거나, 적절한 타입 제약(extends) 없이 복잡한 객체 조작을 하려고 시도하면 컴파일러가 속성을 추론하지 못해 빌드가 깨집니다. 제네릭 매개변수의 필수 역할을 좁히기 위해 적절한 타입 제약 조건을 활용하는 실무 습관이 중요합니다."
*   **매개변수 다형성 스펙**:
    *   **PhD_Book_Author의 학술 스펙**: "TypeScript의 제네릭은 함수나 클래스가 다양한 데이터 타입에 걸쳐 동작할 수 있도록 설계된 **매개변수 다형성(Parametric Polymorphism)**을 구현합니다. C++ 템플릿과 같이 각 타입 인스턴스마다 컴파일 타임에 물리적으로 새로운 기계어 코드를 생성해 내는 단형성화(Monomorphization) 방식과 달리, TypeScript 제네릭은 컴파일 타임에 타입 정보를 엄격히 검증한 후 자바스크립트 변환 시 모두 제거(Type Erasure)하여 런타임에 단 하나의 공유된 함수 객체로만 동작하게 만듭니다."

## 🧪 실습 미션
*   **미션 파일**: `mission.ts`
*   **요구 사항**:
    1.  `wrapInContainer<T>(value: T)`: 입력받은 값을 제네릭 타입 `T`로 캡슐화한 컨테이너 객체를 반환합니다.
        - 반환값 형태는 `{ data: T; timestamp: number }`여야 합니다.
        - `data` 속성에는 입력받은 `value`를 매핑하고, `timestamp`에는 현재 시간 밀리초(number)를 기록합니다.
        - 함수 시그니처에 제네릭 타입 매개변수 `T`가 명확히 결합되어 입력 타입 정보가 반환 타입으로 완벽히 전파되도록 구성해야 합니다.
        - `timestamp` 값은 `Date.now()`를 활용해 생성합니다.

## 💡 힌트 및 트러블슈팅
*   **제네릭 타입 반환**: 반환될 객체 구조 `{ data: T; timestamp: number }`에 맞춰 함수의 반환 타입을 지정합니다. TypeScript는 함수 내부의 return 코드를 통해 타입을 추론하므로 `wrapInContainer<T>(value: T)` 형태로만 선언하고 구현하면 컴파일러가 반환형을 알아서 추론합니다.
*   **동적 타입 보존**: `wrapInContainer`에 어떤 데이터를 인수로 넘기든, 반환된 객체의 `data` 속성을 접근할 때 해당 인수의 고유한 메서드(예: 문자열인 경우 `.substring()`, 객체인 경우 특정 프로퍼티 등)가 정상적으로 코드 완성 목록에 나타나는지 편집기에서 직접 관찰해 볼 수 있습니다.
