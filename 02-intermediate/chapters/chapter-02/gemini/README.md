# Chapter 02: TypeScript 기본 타입 바인딩 (Basic Type Binding)

## 📖 핵심 개념
TypeScript의 핵심인 다양한 변수 및 객체, 배열의 정적 타입 지정 방법을 이해하고 Union 및 Intersection 타입을 활용해 봅니다.

*   **기본 타입 바인딩 (Type Binding)**:
    *   변수나 매개변수 선언 시 콜론(`:`) 뒤에 명시하는 정적 타입 규격입니다. (예: [용어 사전](../../../../GLOSSARY.md#union-intersection-types) 참고)
    *   **Friendly_Tutor의 친절한 비유**: "상자에 담을 수 있는 물건의 종류(타입)를 상자 표면에 라벨지로 붙여서 내부 물건을 보증하는 것과 같습니다. '과일 상자' 라벨이 붙은 상자엔 오직 과일만 넣을 수 있습니다."
*   **유니온 타입 (Union Type)**:
    *   두 개 이상의 타입 중 하나가 될 수 있는 타입을 지정하는 방식입니다 (`|` 연산자 사용).
    *   **Senior_Practitioner의 실무 팁**: "Union 타입을 사용할 때 적절한 타입 가드(Type Guarding / Narrowing)를 거치지 않고 객체의 프로퍼티나 메서드에 접근하려고 하면 컴파일러 단계에서 즉시 에러가 발생하므로 타입 좁히기 연산이 필수적입니다."
*   **서브타이핑 관계**:
    *   **PhD_Book_Author의 학술 스펙**: "TypeScript는 명시적인 상속 관계에 의존하는 명목적 타이핑(Nominal Typing) 대신, 객체의 형태와 구조를 기준으로 타입 호환성을 판단하는 **구조적 타이핑(Structural Typing)** 규칙을 채택합니다. 따라서 어떤 객체가 선언된 타입의 필수 속성을 모두 가지고 있다면 명시적 타입 이름과 상관없이 서브타입 관계가 성립되어 호환성을 갖추게 됩니다."

## 🧪 실습 미션
*   **미션 파일**: `mission.ts`
*   **요구 사항**:
    1.  `formatUser(user: { id: number; name: string; role: 'admin' | 'user' })`: 유저 객체를 매개변수로 받아 지정된 포맷의 문자열을 반환합니다.
        - 반환 형식은 `"[ROLE] name (ID: id)"` 형태입니다. 단, `ROLE`은 대문자로 변환하여 출력되어야 합니다.
        - 매개변수의 각 필드에 대해 타입 바인딩이 정확하게 명시되어야 합니다.

## 💡 힌트 및 트러블슈팅
*   **문자열 템플릿 리터럴**: 포맷팅된 문자열을 결합할 때 ES6 템플릿 리터럴(`` `[${user.role.toUpperCase()}] ...` ``)을 활용하면 코드가 훨씬 직관적이고 가독성이 향상됩니다.
*   **리터럴 타입 매칭**: `role` 속성은 `'admin'` 또는 `'user'` 두 가지 문자열 값만 허용하는 유니온 리터럴 타입입니다. 이를 표준 대문자 리터럴(예: `'ADMIN'`, `'USER'`)로 출력하기 위해 내장 메서드인 `.toUpperCase()`를 안전하게 호출할 수 있는 흐름을 만드세요.
