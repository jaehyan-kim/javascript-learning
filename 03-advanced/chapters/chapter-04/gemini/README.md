# Chapter 04: React 성능 최적화: `useMemo`, `useCallback`, `React.memo` 튜닝

## 📖 핵심 개념
연산 비용이 큰 함수의 결과값이나 함수 인스턴스를 메모리에 보존(Memoization)하여, 불필요한 재연산과 하위 컴포넌트의 비효율적인 리렌더링을 방지하는 성능 최적화 기법입니다.

*   **메모이제이션 (Memoization)과 참조 무결성**:
    *   동일한 입력에 대해 이전 계산 결과를 메모리에 저장하고 재사용하여 실행 속도를 높이는 최적화 기법입니다. (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#usememo-usecallback) 참고)
    *   **Friendly_Tutor의 친절한 비유**: "자주 물어보는 어려운 수학 문제의 답안(연산 결과)을 칠판 한 구석에 미리 적어놓고(useMemo), 똑같은 질문을 받을 때마다 암산 없이 칠판을 보고 즉시 정답을 말하는 것과 같습니다."
*   **얕은 비교와 무의미한 튜닝 버그**:
    *   객체나 배열의 최상위 프로퍼티만 1단계 수준에서 동일한 주소값이나 원시값을 가지는지 대조하는 비교 연산입니다. (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#react-memo-shallow-comparison) 참고)
    *   **Senior_Practitioner의 실무 팁**: "자식 컴포넌트에 `React.memo`를 걸어두었더라도, 부모 컴포넌트 내부에서 생성한 객체나 인라인 함수를 `useMemo`나 `useCallback` 없이 그냥 넘겨주면 매 렌더링마다 새로운 참조 주소가 생성됩니다. 이로 인해 자식의 얕은 비교가 매번 실패하여 최적화 효과가 0이 되고, 오히려 비교 연산 비용만 추가되는 '무의미한 튜닝 버그'가 발생하므로 반드시 참조 무결성을 유지해 주어야 합니다."
*   **참조 투명성 (Referential Transparency)**:
    *   프로그램의 표현식을 그 표현식의 결과값으로 대체해도 전체 프로그램의 동작이 변하지 않는 함수형 프로그래밍 성질입니다.
    *   **PhD_Book_Author의 학술 스펙**: "리액트의 렌더링 파이프라인은 가상 DOM을 생성하는 렌더 단계(Render Phase)와 실제 변경 사항을 DOM에 반영하는 커밋 단계(Commit Phase)로 나뉩니다. 참조 투명성을 가지는 순수 함수에 메모이제이션을 적용하면 렌더 단계에서의 불필요한 호출 비용을 대폭 줄일 수 있으며, JavaScript 렉시컬 환경에 인자값과 결과값을 캐싱함으로써 불필요한 컴포넌트 트리 하향식 평가 연산을 O(1) 수준으로 단축시킵니다."

## 🧪 실습 미션
*   **미션 파일**: `mission.ts`
*   **요구 사항**:
    1.  `memoize<T>(computeFn: (...args: any[]) => T): (...args: any[]) => T`: 함수 연산 결과를 렉시컬 클로저 공간에 캐싱하여, 동일한 인자값으로 호출 시 본래의 연산 없이 즉시 이전 캐시값을 반환하는 고차 메모이제이션 함수를 구현하십시오.
        - 매 호출 시 전달된 인자 배열(`args`)을 이전 호출의 인자들과 비교(`Object.is` 기준)하여 모두 동일하다면 계산을 생략하고 저장된 결과값을 반환해야 합니다.
        - 인자가 달라진 경우에는 원래 함수 `computeFn`을 재실행하여 결과를 갱신하고 인자와 결과 캐시를 업데이트해야 합니다.

## 💡 힌트 및 트러블슈팅
*   이전 인자들과 현재 인자들의 개수가 다르거나, 어느 하나라도 `Object.is` 비교 결과가 다르다면 인자가 변경된 것으로 판별합니다.
*   클로저를 활용해 반환 함수 외부 영역에 `lastArgs`와 `lastResult`, 그리고 최초 실행 여부를 판단할 `hasRun` 변수를 선언하여 상태를 기억하게 만드십시오.
