# Chapter 10: Hydration 동작 원리와 최적화 및 런타임 Hydration Mismatch 디버깅 (Hydration Mismatch Debugging)

## 📖 핵심 개념
서버 사이드 렌더링(SSR)을 통해 미리 브라우저에 내려진 정적 HTML 마크업 위에 React 런타임이 로드되어 가상 DOM을 재구성하고, 두 구조를 비교하여 이벤트 리스너를 결합하는 하이드레이션(Hydration)의 동작 메커니즘과 서버-클라이언트 불일치 오류(Hydration Mismatch)를 탐지하고 디버깅하는 원리입니다.

*   **하이드레이션 (Hydration)**:
    *   서버의 정적 마크업을 지우지 않고 클라이언트 React 가상 DOM의 인터랙티브 바인딩을 적용하는 초기 결합 프로세스입니다. (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#hydration) 참고)
    *   **Friendly_Tutor의 친절한 비유**: "인쇄 공장에서 완성되어 온 일러스트 삽화 책(서버 HTML)이 있고, 그 위에 딱 들어맞는 투명 필름지 글판(클라이언트 VDOM)을 겹쳐 대조해 붙이는 작업과 같습니다. 두 책의 삽화 크기와 글판의 위치가 정확히 일치하면 완벽히 작동하는 책(인터랙티브한 상태)이 되지만, 한 글자라도 오탈자나 줄 위치가 어긋나면(Mismatch) 책을 찢고 다시 인쇄해야 하는 낭비가 발생합니다."
*   **런타임 하이드레이션 mismatch 디버깅**:
    *   서버에서 만들어진 마크업과 브라우저에서 처음 생성한 가상 DOM이 다를 때 브라우저 돔 트리가 파괴되고 깜빡이는 현상(Flickering)이 발생하며 사용자 경험을 훼손합니다.
    *   **Senior_Practitioner의 실무 팁**: "가장 빈번한 mismatch 원인은 서버 렌더링 시점의 현재 시각(`new Date()`)이나 랜덤값(`Math.random()`)을 방어 처리(useEffect 이후 호출 등) 없이 렌더링에 그대로 노출할 때 발생합니다. 또한 `window` 객체 유무로 서버/클라이언트의 HTML을 다르게 렌더링하면 브라우저는 초기 트리를 강제로 파괴 후 전체 재렌더링하므로, Mismatch 발생 지점(태그, 속성, 텍스트)과 구조적 원인을 명확히 역추적하여 감지하는 디버깅 도구를 적용하는 것이 필수적입니다."
*   **React hydrateRoot 비교 알고리즘**:
    *   **PhD_Book_Author의 학술 스펙**: "리액트의 `hydrateRoot` 함수는 돔 노드를 순회하며 클라이언트 가상 DOM 노드와 1대1 매칭 및 트리 비교(Reconciliation)를 재귀적으로 수행합니다. 태그의 정합성, 속성(Attribute)의 일치 여부, 텍스트 노드의 문자열 정밀 일치를 판단하며, 불일치가 탐지될 경우 런타임에 에러 경고(`Warning: Did not expect server HTML to contain...`)를 콘솔에 출력하고 mismatch 노드부터 하위 트리 전체를 새로 생성한 클라이언트 DOM 노드로 파괴 및 대체(Patch)하는 브라우저 런타임 돔 패치 메커니즘을 내포하고 있습니다."

## 🧪 실습 미션
*   **미션 파일**: `mission.ts`
*   *(참고: 이 미션은 실제 브라우저 DOM API를 사용하지 않고 단순화된 가상 파서로 하이드레이션 불일치 탐지 로직만 구현하며, 실제 DOM 파괴나 재생성, 이벤트 재바인딩은 구현 범위에 포함되지 않습니다.)*
*   **요구 사항**:
    1.  `detectHydrationMismatch(serverHtml: string, clientDom: VirtualNode)`:
        - 서버로부터 받은 HTML 원본 문자열과 브라우저의 React 최초 가상 돔(`VirtualNode`) 구조를 재귀적으로 대조하는 정밀 mismatch 디버깅 엔진을 구현합니다.
        - 비교해야 하는 불일치 유형은 **태그명 불일치**(`TAG_MISMATCH`), **속성 불일치**(`ATTR_MISMATCH`), **텍스트 내용 불일치**(`TEXT_MISMATCH`), **자식 노드 개수 불일치**(`CHILD_COUNT_MISMATCH`)의 4가지입니다.
        - 불일치가 발생한 노드의 경로를 부모 태그부터 꺾쇠(`>`) 기호로 구분하여 반환 객체의 `path`에 기록해야 합니다. (예: `div > p > text` 또는 속성의 경우 `div (className)`)

## 💡 힌트 및 트러블슈팅
*   HTML 원본 문자열을 가상 돔 구조(`VirtualNode` 타입)로 손쉽게 다루기 위해, 태그명, 속성(class 속성은 className으로 매핑), 자식 배열을 분리해내는 경량 HTML 파서 헬퍼를 먼저 구현하여 서버 DOM 트리를 구축하세요.
*   구축된 서버 DOM 트리와 매개변수로 들어온 `clientDom` 트리 구조를 루트 노드부터 시작하여 재귀적인 비교 함수를 만들어 대조하세요.
*   문자열 텍스트 노드의 양끝 공백이나 줄바꿈은 브라우저 렌더링 시 의미가 없으므로 `trim()` 메서드를 이용해 공백 텍스트는 필터링한 후 자식 요소로 등록하여 비교 정확도를 높여야 합니다.
