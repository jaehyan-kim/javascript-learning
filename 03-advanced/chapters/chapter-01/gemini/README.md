# Chapter 01: 가상 DOM 렌더링 조정(Reconciliation) 및 Diffing 알고리즘 설계

## 📖 핵심 개념
가상 DOM의 변경 사항을 최소한의 실제 DOM 조작으로 반영하기 위해, 두 개의 가상 DOM 트리를 비교하여 차이점(Patch)을 계산해 내는 알고리즘입니다.

*   **가상 DOM (Virtual DOM)**:
    *   실제 브라우저의 DOM 트리를 자바스크립트 객체 형태로 가볍게 모제화하여 표현한 애플리케이션 추상화 레이어입니다. (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#virtual-dom) 참고)
    *   **Friendly_Tutor의 친절한 비유**: "두 한글 문서의 원본과 개정본을 한 줄 한 줄 대조하여 빨간 펜으로 추가, 삭제, 수정해야 할 부분만 표시해 두는 교정(Patch) 작업과 같습니다."
*   **Key 속성과 리스트 렌더링**:
    *   리스트 내 요소를 고유하게 식별할 수 있도록 부여하는 고유값 식별자입니다.
    *   **Senior_Practitioner의 실무 팁**: "배열의 `index`를 `key`로 사용하면 리스트 중간에 요소를 삽입하거나 삭제할 때 React가 DOM 노드를 재사용하지 못하고 하위 노드를 전부 파괴 후 재생성하게 됩니다. 이로 인해 입력 중이던 `<input>`의 포커스가 유실되거나 상태가 엉뚱한 노드로 전이되는 버그와 심각한 렌더링 성능 저하가 발생하므로, 반드시 데이터의 고유한 ID(예: `id`)를 key로 지정해야 합니다."
*   **Diffing Heuristics**:
    *   일반적인 트리 비교 O(n^3) 알고리즘을 휴리스틱 가정을 통해 O(n)으로 단축시키는 리액트의 알고리즘 규칙입니다.
    *   **PhD_Book_Author의 학술 스펙**: "React의 조정(Reconciliation) 알고리즘은 두 가지 가정을 기반으로 구현되었습니다. 첫째, 서로 다른 타입의 두 엘리먼트는 서로 다른 트리를 만들어낸다. 둘째, 개발자가 `key` 프로퍼티를 제공함으로써 여러 렌더링 사이에서 어떤 자식 엘리먼트가 안정적인지 힌트를 줄 수 있다. 이 휴리스틱 덕분에 루트 노드부터 시작하여 깊이 우선 탐색(DFS)을 돌며 동시 자식 노드 순회를 수행함으로써 선형 시간 O(n) 만에 두 트리의 변경 목록을 연산해 낼 수 있습니다." (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#reconciliation-diffing-heuristics) 참고)

## 🧪 실습 미션
*   **미션 파일**: `mission.ts`
*   **요구 사항**:
    1.  `diffNodes(oldNode: VirtualNode, newNode: VirtualNode): Patch[]`: 두 가상 노드를 비교하여 속성 변경, 자식 교체, 노드 추가/삭제 패치 연산 목록을 추려 반환하는 핵심 비교 엔진을 구현하십시오.
        - 노드 타입이나 태그가 다르면 `REPLACE` 패치를 생성해야 합니다.
        - 두 노드가 모두 문자열(텍스트 노드)인 경우 내용이 다르면 `TEXT` 패치를 생성합니다.
        - 노드 타입이 같으면 속성(Props)의 변경(추가/수정/삭제)을 감지하여 `PROPS` 패치를 생성합니다.
        - 자식 노드들을 비교하여 추가, 삭제, 수정을 반영하는 `CHILDREN` 패치 목록(`ChildPatch`)을 생성해야 합니다. 이때 `key`를 기준으로 매칭하여 최소한의 업데이트를 계산합니다. (단, 이 실습에서는 노드가 이동(재정렬)하는 경우의 MOVE 최적화 처리를 생략하고 단순화하여 구현합니다.)

## 💡 힌트 및 트러블슈팅
*   자식 노드 비교 시 `key`가 있는 노드와 없는 노드가 혼재할 수 있습니다. `key`가 존재한다면 `key`가 같은 노드끼리 비교하여 `UPDATE` 패치를 수행하고, `key`가 존재하지 않는다면 동일 인덱스의 노드끼리 비교합니다.
*   Props 비교 시 이전 객체에는 존재하지만 새 객체에는 없는 키는 `removed`에, 값이 변경되었거나 새로 추가된 키는 `added`에 담아 `PROPS` 패치로 보고합니다.
