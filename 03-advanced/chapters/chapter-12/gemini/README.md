# Chapter 12: AI LLM API 연동 및 스트리밍 답변 화면 실시간 렌더링 (AI LLM Stream Rendering)

## 📖 핵심 개념

AI LLM API가 생성하는 답변 토큰을 서버로부터 실시간으로 전달받는 SSE(Server-Sent Events) 스트리밍 기술과, 이를 브라우저 화면에 끊김 없이 타이핑하듯 렌더링하고 사용자 경험(UX)을 극대화하는 프론트엔드 연동 기법입니다.

- **SSE 스트리밍 토큰 (Server-Sent Events)**:
  - 서버에서 클라이언트로 단방향 비동기 이벤트를 지속해서 스트리밍하는 HTTP 표준 기술입니다. (자세한 내용은 [용어 사전](../../../../GLOSSARY.md#sse-streaming) 참고)
  - **Friendly_Tutor의 친절한 비유**: "길고 긴 편지 한 통을 상대방이 다 쓸 때까지 멍하니 몇 분 동안 기다려서 한 번에 받는 것(전체 대기) 대신, 상대방이 타자기(SSE 스트리밍)로 한 자 한 자 입력할 때마다 내 눈앞의 종이에 실시간으로 글자가 찍혀 즉시 내용을 읽어 내려가는 방식과 같습니다. 단 한 단어만 도착해도 즉각 피드백을 받을 수 있습니다."
- **마크다운 토큰 잘림 예외 처리 및 UX 개선**:
  - 네트워크 패킷 단위로 쪼개져서 유입되는 LLM 답변 조각에서 발생할 수 있는 데이터 파싱 및 레이아웃 결함을 방지하는 설계입니다.
  - **Senior_Practitioner의 실무 팁**: "LLM 스트리밍 중에는 마크다운 태그 기호(예: ` ``` `와 같은 코드 블록 선언부)가 여러 청크에 걸쳐 쪼개진 채 수입될 수 있습니다(예: 첫 청크에서 ` `` `, 다음 청크에서 `` ` ``). 이를 버퍼링 없이 즉시 마크다운 파서에 통과시키면 닫히지 않은 코드 블록 때문에 레이아웃이 비정상적으로 깨집니다. 따라서 유입된 라인의 불완전한 버퍼를 라인 단위(`\n`)로 온전히 수집할 때까지 홀드해두는 스트림 정렬 기법을 적용해야 합니다. 또한, 텍스트가 추가되어 스크롤 바깥으로 넘어가기 전에 사용자 화면의 스크롤을 실시간으로 화면 하단에 고정하는(자동 스크롤) 코드도 포함해야 완성도 높은 UX를 구현할 수 있습니다."
- **EventSource API 및 SSE 프로토콜 헤더 명세**:
  - **PhD_Book_Author의 학술 스펙**: "EventSource 또는 Fetch API를 통한 SSE 수신 시, 서버는 응답 헤더의 `Content-Type`을 `text/event-stream`으로 설정하고, 캐싱을 방지하기 위해 `Cache-Control: no-cache`를 세팅해야 합니다. 전송되는 각 데이터 블록은 `data: ` 프리픽스로 시작하여 개행 문자 두 개(`\n\n`)로 끝나며, 하나의 이벤트 스트림이 종료되면 `data: [DONE]`과 같은 관례적인 종단 토큰을 수신하게 됩니다. 브라우저는 이를 스트림 버퍼링 없이 즉각 클라이언트 런타임의 큐로 인입시켜 메인 스레드에 이벤트를 전파합니다."

## 🧪 실습 미션

- **미션 파일**: `mission.ts`
- **요구 사항**:
  1.  `renderLlmStream(streamResponse: ReadableStream<Uint8Array>, domElement: HTMLElement): Promise<void>`:
      - 전달받은 `streamResponse`로부터 바이너리 청크를 읽어들여 텍스트로 디코딩합니다.
      - 수신된 데이터 문자열은 SSE 라인들로 분리 처리합니다. 즉, `\n`을 기준으로 잘라내며, 청크 끝부분에 걸쳐 완전히 완성되지 않은 미완성 라인은 잔여 버퍼에 유지했다가 다음 청크와 병합해 복원해야 합니다.
      - 파싱된 라인 중 `data: `로 시작하는 라인을 찾아 데이터를 추출합니다.
      - 만약 데이터 내용이 `[DONE]`이면 스트림 수집 루프를 종료합니다.
      - 데이터 내용이 JSON 포맷인 경우(예: `{"token": "..."}` 또는 `{"choices":[{"delta":{"content":"..."}}]}`) 파싱하여 토큰 문자열을 획득하고, JSON이 아닐 경우(예: 일반 텍스트 문자열) 원본 그대로 텍스트 조각으로 활용합니다.
      - 획득한 텍스트 조각을 `domElement`에 즉시 누적하여 반영하고, 사용자가 읽기 편하도록 `domElement.scrollTop = domElement.scrollHeight`를 수행해 실시간 스크롤 하단 고정 피드백을 구현합니다.

## 💡 힌트 및 트러블슈팅

- 테스트 코드에서 DOM 환경을 사용하기 위해 `mission.test.ts` 파일의 최상단에 `/** @jest-environment jsdom */` 주석 지시어를 반드시 명시해 주십시오.
- 버퍼에 남아 있는 미완성 라인은 스트림이 완전히 닫혔을 때(`done: true`) 디코더를 최종 정리(`stream: false`)하고 남은 데이터를 최종 파싱하여 화면에 뿌려주는 처리가 필요합니다.
- JSON 문자열을 파싱할 때 `JSON.parse`가 던지는 에러를 잡는 `try-catch` 안전장치를 마련하여, 예기치 못한 원본 텍스트 유입 시에도 전체 렌더링이 중단되지 않고 텍스트 그대로 렌더링되게 설계해야 현업에서 발생하는 파싱 결함을 차단할 수 있습니다.
- *(보안 경고): 신뢰할 수 없는 외부 소스(LLM 응답 등)의 텍스트를 파싱하여 DOM 요소에 누적할 때는 XSS(교차 사이트 스크립팅) 공격을 방지하기 위해 `innerHTML` 대신 `textContent`를 사용하거나, 불가피할 경우 엄격한 이스케이프 처리를 수행해야 합니다.*
